Multi-Tenancy Architecture

All four SaaS products are built on a strict multi-tenant architecture. Each customer's data is logically isolated with no cross-tenant access possible under any operating condition:

  • API Layer: Tenant-scoped JWT claims enforced at the API gateway — every request is validated against the originating tenant identity
  • Data Layer: Row-level security (RLS) or dedicated schema-per-tenant at the database layer, chosen based on data sensitivity
  • Credentials: Integration API keys and OAuth tokens stored per-tenant in an encrypted secrets vault (AWS Secrets Manager / Azure Key Vault)
  • Audit Trail: Tenant provisioning and de-provisioning are automated and generate immutable audit logs

Data Governance & Compliance

Enterprise-grade data protection as standard practice — not a premium add-on. Every client engagement is governed by multi-tenant isolation, regional data residency, signed Data Processing Agreements, and role-based access controls that are audited continuously.

Multi-Tenancy by Service Line

Managed Services & IT Support

Client environments are accessed via dedicated, credential-isolated VPN/bastion connections. No shared credentials or monitoring agents span across clients. Each client's environment is a fully segregated operational boundary.

SaaS Products

Tenant isolation enforced at two layers: tenant-scoped JWT claims at the API gateway, and row-level security (RLS) or dedicated schemas at the database layer. Integration credentials stored per-tenant in an encrypted secrets vault. Automated provisioning with immutable audit trails.

Accounting & Bookkeeping

Each client's financial data is maintained in a fully isolated data store or schema. Accounting software API keys and OAuth tokens are stored per-tenant in an encrypted secrets vault and never shared across clients.

E-commerce Platform Services

Per-merchant access tokens, store credentials, and analytics data are scoped to each merchant and never commingled. Platform API credentials are rotated and audited regularly.

Data Governance Practices

Data Classification Policy

All data is classified at ingestion as Public, Internal, Confidential, or Restricted. Client business data is Confidential at minimum; financial records and PII are Restricted — subject to encryption at rest and in transit, vault-based credential storage, and immutable audit logging.

Data Residency Controls

Client data is stored in the geographic region agreed in the service contract. Compliant with India DPDP Act 2023 (Indian clients), Australia Privacy Act 1988 / Australian Privacy Principles (Australian clients), and UAE Personal Data Protection Law — PDPL (Middle East clients).

Data Processing Agreements (DPAs)

A signed DPA is executed with every client before any data ingestion. DPAs cover data classification, residency, retention, deletion, breach notification obligations, and sub-processor disclosures — compliant with Australian Privacy Principles, UAE PDPL, and GDPR where applicable.

RBAC & Audit Logging

Role-based access control (RBAC) is enforced across all internal systems. Staff access to client data is on a strict need-to-know basis, reviewed quarterly. All access events are centrally logged via SIEM and retained as immutable records for a minimum of 12 months.

Data Retention & Deletion

Defined retention periods apply per data classification. Automated archival and deletion pipelines enforce these periods. Client data is purged within 30 days of contract termination upon written request, with a deletion certificate issued.

Compliance Certifications Roadmap

  • ISO 27001: Targeted within 18 months of operations — required for credibility with Australian and GCC clients
  • SOC 2 Type II: Evaluation within 24 months, covering Security, Availability, and Confidentiality trust service criteria — directly validates multi-tenancy and data governance controls
  • India DPDP Act 2023: Compliant from Day 1 for all Indian client and employee personal data
  • Australia Privacy Act 1988 (APPs): Full compliance for all Australian client data
  • UAE PDPL: Compliance for all Middle East client data
